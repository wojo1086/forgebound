Getting Started
Quickstart
From zero to adventurer in three API calls. All endpoints use JSON bodies and return JSON responses.
Create an Account
Register with an email and password. This creates your account in the Forgebound world.
Request
POST /api/auth/register

{
  "email": "hero@example.com",
  "password": "your-secret-password"
}
Log In
Authenticate and receive an access token. Include this token as a Bearer token in the Authorization header for all authenticated endpoints.
Request
POST /api/auth/login

{
  "email": "hero@example.com",
  "password": "your-secret-password"
}
Response
{
  "accessToken": "eyJhbG...",
  "refreshToken": "v1.MDA...",
  "expiresIn": 3600
}
Create a Character
Choose a race, a class, and distribute 27 points across six ability scores using D&D 5e point buy rules. The server applies racial bonuses and computes your HP and AC.
Request
POST /api/characters
Authorization: Bearer <token>

{
  "name": "Thorin Ironfist",
  "raceId": "dwarf",
  "classId": "warrior",
  "strength": 15,
  "dexterity": 10,
  "constitution": 14,
  "intelligence": 8,
  "wisdom": 12,
  "charisma": 8
}
Authentication
Auth
Register and authenticate players. Authenticated endpoints require a Bearer token in the Authorization header.
POST /api/auth/register

Create a new player account.

ParameterTypeDescription
emailrequired string Valid email address
passwordrequired string 8–128 characters
Request
POST /api/auth/register
Content-Type: application/json

{
  "email": "hero@example.com",
  "password": "your-secret-password"
}
Response 200
{
  "id": "550e8400-e29b-...",
  "email": "hero@example.com",
  "message": "Account created. Please check your email and confirm your address before logging in."
}
POST /api/auth/login

Authenticate and receive access and refresh tokens.

ParameterTypeDescription
emailrequired string Your email
passwordrequired string Your password
Response 200
{
  "accessToken": "eyJhbGciOi...",
  "refreshToken": "v1.MDAyN...",
  "expiresIn": 3600,
  "user": {
    "id": "550e8400-e29b-...",
    "email": "hero@example.com"
  }
}
Use accessToken as Authorization: Bearer <token> for all authenticated endpoints.
GET /api/auth/me Auth

Get the currently authenticated player's profile.

Response 200
{

  "email": "hero@example.com",
  "createdAt": "2026-03-27T12:00:00Z"
}

Characters
Character Creation
Create your adventurer. Each account can have one character. Ability scores use D&D 5e point buy — 27 points, scores from 8 to 15.
POST /api/characters Auth

Create a new character. Ability scores are base values before racial bonuses. The server validates point buy cost, applies racial bonuses, and computes derived stats.

raceIdrequired string human elf dwarf halfling orc
classIdrequired string warrior mage rogue cleric ranger
strengthrequired int 8–15 (before racial bonus)
dexterityrequired int 8–15
constitutionrequired int 8–15
intelligencerequired int 8–15
wisdomrequired int 8–15
charismarequired int 8–15

POST /api/characters

World Map
Explore the World
The world is a 100×100 grid of terrain cells. Towns and landmarks are always visible. Hidden points of interest are revealed when your character walks onto them. View the interactive map →
GET /api/map
Returns the full terrain grid and all visible POIs (towns & landmarks). If you include an auth token, your discovered hidden POIs are included too.
Auth: Optional
Response
{
  "width": 50,
  "height": 50,
  "terrain": [["ocean", "ocean", ...], ...],
  "pois": [
    {
      "id": "thornhaven",
      "name": "Thornhaven",
      "type": "town",
      "category": "town",
      "x": 25,
      "y": 24,
      "terrain": "plains",
      "description": "A bustling trade hub..."
    }
  ],
  "discoveries": []
}
GET /api/map/cell/:x/:y
Returns terrain and POI info for a specific cell. Useful for checking what’s at a coordinate without downloading the whole map.
Auth: Optional
Example: GET /api/map/cell/25/24
{
  "x": 25,
  "y": 24,
  "terrain": "plains",

    "id": "thornhaven",

Travel
Travel & Movement
Move your character across the world. Travel takes real time based on terrain difficulty. You can step one cell at a time or pathfind to any reachable coordinate. Hidden POIs are discovered as you pass through cells.
POST /api/travel/move
Step one cell in a cardinal direction. Travel time depends on the destination terrain. You cannot move while already traveling.
Auth: Required
Request
{
  "direction": "north"
}
Response
{
  "direction": "north",
  "from": { "x": 50, "y": 50 },
  "destination": { "x": 50, "y": 49 },
  "terrain": "plains",
  "travelSeconds": 10,
  "startedAt": "2026-03-27T12:00:00.000Z",
  "eta": "2026-03-27T12:00:10.000Z"
}
Valid directions: north, south, east, west
POST /api/travel/go
Travel to a specific coordinate. The server pathfinds the shortest route using A* and terrain costs. Total travel time is the sum of all cells along the path.
Auth: Required
Request
{
  "x": 76,
  "y": 48
}
Response
{
  "from": { "x": 50, "y": 50 },
  "destination": { "x": 76, "y": 48 },
  "path": [{ "x": 50, "y": 50 }, ...],
  "totalCells": 28,
  "travelSeconds": 385,
  "startedAt": "2026-03-27T12:00:00.000Z",
  "eta": "2026-03-27T12:06:25.000Z"
}
GET /api/travel/status
Check your current travel progress. If the ETA has passed, the server resolves the trip—updating your position and returning any POIs discovered along the way.
Auth: Required
Response (traveling)
{
  "isTraveling": true,
  "position": { "x": 50, "y": 50 },
  "estimatedCurrent": { "x": 58, "y": 50 },
  "destination": { "x": 76, "y": 48 },
  "progressIndex": 8,
  "totalCells": 28,
  "startedAt": "...",
  "eta": "..."
}
Response (arrived)
{
  "isTraveling": false,
  "position": { "x": 76, "y": 48 },
  "discoveries": [
    {
      "id": "hidden-shrine-04",
      "name": "Overgrown Shrine",
      "type": "shrine",
      "x": 62, "y": 49
    }
  ]
}
POST /api/travel/cancel
Cancel active travel. Your character stops at the furthest cell reached based on elapsed time. Any hidden POIs along the traversed portion are still discovered.
Auth: Required
Response
{
  "cancelled": true,
  "position": { "x": 58, "y": 50 },
  "cellsTraversed": 8,
  "discoveries": []
}

Inventory
Items & Equipment
Manage your character’s inventory. Pick up items, equip weapons and armor, use consumables like potions and spell scrolls. Equipment modifies your effective stats. Weight is tracked against your carry capacity (based on Strength).
GET /api/inventory
Returns your full inventory: all items, equipped gear, effective stats with bonuses, total weight, and carry capacity.
Auth: Required
Response
{
  "items": [
    {
      "id": "iron-sword",
      "name": "Iron Sword",
      "type": "weapon",
      "rarity": "common",
      "quantity": 1,
      "equipped": true,
      "slot": "weapon"
    }
  ],
  "equipment": {
    "weapon": { "id": "iron-sword", "name": "Iron Sword" },
    "armor": null,
    "helmet": null,
    ...
  },
  "effectiveStats": { "strength": 12, "dexterity": 10, ... },
  "weight": 5,
  "carryCapacity": 150
}
POST /api/inventory/pick-up
Add an item to your inventory. Items are defined in the game data catalog. Checks weight capacity before adding.
Auth: Required
Request
{
  "itemId": "health-potion",
  "quantity": 3
}
Response
{
  "picked_up": "Health Potion",
  "quantity": 3,
  "totalWeight": 1.5,
  "carryCapacity": 150
}
POST /api/inventory/drop
Remove an item from your inventory. Cannot drop equipped items — unequip first.
Auth: Required
Request
{
  "itemId": "health-potion",
  "quantity": 1
}
POST /api/inventory/equip
Equip an item from your inventory. The item must be in your inventory and the correct type for the slot. Equipping to an occupied slot automatically unequips the previous item.
Auth: Required
Request
{
  "itemId": "iron-sword",
  "slot": "weapon"
}
Slots: weapon, armor, helmet, shield, leggings, boots, gloves, ring1, ring2, amulet
POST /api/inventory/unequip
Unequip an item from a slot, returning it to your inventory.
Auth: Required
Request
{
  "slot": "weapon"
}
POST /api/inventory/use
Use a consumable item. Potions restore HP or mana, spell scrolls teach you new spells permanently. The item is consumed on use.
Auth: Required
Request
{
  "itemId": "health-potion"
}
Response
{
  "used": "Health Potion",
  "effect": "Restored 25 HP. HP: 38/38."
}
Consumable types: health potions (restore HP), mana potions (restore mana), spell scrolls (learn a spell)
Spells
Magic & Spellcasting
Your character starts with class-specific spells and can learn more as they level up. Spells at level 1–3 can be learned by study; level 4–5 spells require spell scrolls (consumable items). Casting costs mana. Some spells are usable outside combat (healing, buffs, teleport), while damage spells are reserved for battle.
GET /api/spells
Returns your spellbook: current mana, max mana, and all learned spells with their details.
Auth: Required
Response
{
  "mana": 80,
  "maxMana": 100,
  "spells": [
    {
      "id": "magic-missile",
      "name": "Magic Missile",
      "school": "evocation",
      "spellLevel": 1,
      "manaCost": 8,
      "effectType": "damage",
      "effectValue": 10,
      "damageType": "arcane",
      "targetType": "enemy",
      "levelRequired": 1,
      "classRestriction": "mage"
    }
  ]
}
POST /api/spells/learn
Learn a new spell by study. Only works for spells at spell level 1–3. Higher-level spells require using a spell scroll item via /api/inventory/use. You must meet the spell’s level requirement and class restriction.
Auth: Required
Request
{
  "spellId": "fireball"
}
Response
{
  "learned": "Fireball",
  "message": "You have learned Fireball!",
  "spell": {
    "id": "fireball",
    "name": "Fireball",
    "school": "evocation",
    "spellLevel": 3,
    "manaCost": 25
  }
}
POST /api/spells/cast
Cast a learned spell. Deducts mana and applies the effect. Healing spells restore HP, buff spells increase stats temporarily, Teleport returns you to town. Damage spells can only be cast in combat. Cannot cast while resting.
Auth: Required
Request
{
  "spellId": "heal"
}
Response
{
  "cast": "Heal",
  "manaCost": 10,
  "effect": { "type": "heal_hp", "value": 25 },
  "result": "Heal restored 25 HP. HP: 38/38. Mana: 70/80"
}
Castable effects: heal_hp, restore_mana, teleport_town, buff_strength, buff_dexterity, buff_constitution, buff_intelligence, buff_wisdom, buff_charisma, buff_max_hp
Rest
Rest & Recovery
Recover HP and mana by resting. Camp anywhere for slow time-based regeneration, or visit a town inn for instant full restoration. While resting, you cannot travel, cast spells, or take other actions. You can stop resting early to collect partial recovery.
POST /api/rest/camp
Set up camp and rest. HP and mana regenerate at 0.5/second (30/minute). Duration auto-calculates to fill HP and mana, capped at 10 minutes. You can specify a custom duration in seconds (10–600). Cannot camp if already at full HP and mana.
Auth: Required
Request (optional duration)
{
  "duration": 60
}
Response
{
  "resting": true,
  "type": "camp",
  "duration": 60,
  "hpRegen": 30,
  "manaRegen": 30,
  "startedAt": "2026-03-28T12:00:00.000Z",
  "until": "2026-03-28T12:01:00.000Z"
}
POST /api/rest/inn
Rest at a town inn for instant full HP and mana restoration. Your character must be at a tile that contains a town POI. No body required.
Auth: Required
Response
{
  "rested": true,
  "type": "inn",
  "hp": 38,
  "maxHp": 38,
  "mana": 100,
  "maxMana": 100,
  "location": "Thornhaven"
}
GET /api/rest/status
Check your current rest progress. If resting, shows elapsed time, HP and mana recovered so far, and when rest completes. If not resting, indicates idle state.
Auth: Required
Response (while camping)
{
  "resting": true,
  "type": "camp",
  "elapsed": 30,
  "hpRecovered": 15,
  "manaRecovered": 15,
  "currentHp": 28,
  "currentMana": 65,
  "until": "2026-03-28T12:01:00.000Z"
}
POST /api/rest/stop
Stop resting early. You keep whatever HP and mana you’ve regenerated so far based on elapsed time. Returns a summary of recovery.
Auth: Required
Response
{
  "stopped": true,
  "elapsed": 30,
  "hpRecovered": 15,
  "manaRecovered": 15,
  "hp": 28,
  "maxHp": 38,
  "mana": 65,
  "maxMana": 100
}
Shops
Town Shops
Each town has a unique shop with themed inventory. Coastal towns sell jewelry and provisions, mountain forges offer heavy arms, and forest apothecaries stock the best potions. Stock is per-player and restocks on a timer. You must be physically at a town to use its shop. Characters start with 100 gold.
GET /api/shops/:townId
Browse a town shop’s inventory. Shows all items for sale with current stock, prices, and your gold balance. Stock restocks automatically after the shop’s restock timer elapses. You must be at the town.
Auth: Required
Example: GET /api/shops/thornhaven
{
  "townId": "thornhaven",
  "shop": "Thornhaven General Store",
  "description": "A well-stocked shop...",
  "restockMinutes": 30,
  "gold": 100,
  "items": [
    {
      "id": "health-potion",
      "name": "Health Potion",
      "type": "consumable",
      "rarity": "common",
      "price": 10,
      "sellPrice": 5,
      "levelRequired": 1,
      "weight": 0.5,
      "stock": 10,
      "maxStock": 10
    }
  ]
}
POST /api/shops/:townId/buy
Buy an item from a town shop. Deducts gold, decrements stock, and adds the item to your backpack. Validates gold, stock, level requirement, class restriction, and carry weight.
Auth: Required
Request
{
  "itemId": "health-potion",
  "quantity": 2
}
Response
{
  "bought": "Health Potion",
  "quantity": 2,
  "totalCost": 20,
  "gold": 80,
  "message": "Purchased 2x Health Potion for 20g."
}
POST /api/shops/:townId/sell
Sell an item from your backpack to a shop. Items sell for half their buy price (rounded down). Equipped items must be unequipped first. Quest items cannot be sold.
Auth: Required
Request
{
  "itemId": "copper-dagger",
  "quantity": 1
}
Response
{
  "sold": "Copper Dagger",
  "quantity": 1,
  "unitPrice": 1,
  "totalPrice": 1,
  "gold": 101,
  "message": "Sold 1x Copper Dagger for 1g."
}
Leveling
Experience & Leveling
Characters earn XP by exploring the world. Discovering hidden POIs, visiting towns for the first time, and simply traveling all grant XP. When enough XP is accumulated, the character levels up (max level 20). On level-up: max HP and max mana increase based on class, the character is fully healed, eligible spells are auto-learned, and 1 unspent stat point is granted.

POST /api/characters/allocate-stat
Spend an unspent stat point to increase an ability score by 1. You earn 1 stat point per level-up. Ability scores cap at 20. Increasing Constitution recalculates max HP; increasing Intelligence recalculates max mana.
Auth: Required
Request
{
  "stat": "constitution"
}
Response
{
  "allocated": "constitution",
  "newValue": 15,
  "unspentStatPoints": 0,
  "message": "Constitution increased to 15.",
  "maxHp": 28,
  "hp": 28
}
Valid stats: strength, dexterity, constitution, intelligence, wisdom, charisma. When allocating Intelligence, the response includes maxMana and mana instead of HP fields.
Combat
Turn-Based Combat
Encounters trigger randomly during travel (based on terrain danger) and at specific POIs like ambush sites. Combat is turn-based: you choose an action, the server resolves your turn and the monster’s turn, and returns the results. On victory you earn XP, gold, and loot. On defeat you lose 10% gold and some XP, then respawn at the nearest town.

Encounter Rates (per cell)

TerrainChance
Plains3%
Coast2%
Forest8%
Desert6%
Swamp10%
Mountain12%

Monster Scaling

Monsters have base level-1 stats that scale with area level. Areas near the center (50,50) are low-level; edges are high-level.

StatFormula
HPbaseHp × (1 + 0.3 × (level-1))
DamagebaseDmg × (1 + 0.2 × (level-1))
ACbaseAc + floor(level / 3)
XP RewardbaseXp × level

}
Actions: attack uses your equipped weapon (STR modifier). cast uses a learned spell (INT/WIS modifier, costs mana). use_item consumes a backpack item (costs your turn). flee has a DEX-based chance (40% base); fails against bosses.
Hit rolls: d20 + modifier vs target AC. Natural 20 = critical (2× damage). Natural 1 = miss.
While in combat: You cannot travel, rest, shop, or change equipment.

Status Effects

Both players and monsters can apply status effects during combat. Spells apply effects on hit (with a chance roll). Rare+ weapons have a chance to proc effects on melee attacks. Monster abilities can also inflict effects. Effects do not stack — reapplying the same effect refreshes its duration.

Sources

SourceApplication
SpellsChance to apply on hit (e.g. Fireball: 50% burn)
WeaponsRare+ weapons have a proc chance (e.g. Flamebrand: 25% burn)
Monster abilitiesMany monsters inflict effects (e.g. Spider: poison, Lich: silence)
Purify spellRemoves all negative effects (DoTs, debuffs, controls)
Processing order: Effects are processed at turn start before actions. DoT damage and regen heal first, then stun/freeze/silence are checked. A stun applied on turn N takes effect on turn N+1.
No stacking: Reapplying the same effect type refreshes the duration; it does not stack multiple instances.
Damage pipeline: Raw damage → Outgoing multiplier (Weakness/Enrage) → Vulnerability (Freeze +50%) → Shield absorption.
Dungeons
Dungeon System
Multi-room instanced dungeons at POI locations. Each dungeon generates random rooms (combat, treasure, trap, rest) with a boss at the end. Progress is saved between sessions. Completed dungeons reset after 24 hours.

Dungeon-Eligible POI Types:

CategoryPOI TypesRooms
Landmarksdungeon, ruins, mine, fortress5–8
Hiddenancient_tomb, secret_cave, smuggler_den3–5

Room Types:

TypeWeightDescription
combat40%Fight a monster themed to the dungeon type
treasure20%Gold + 2 loot rolls (rarity-weighted)
trap20%DEX-based avoidance check. 15% max HP damage on fail
rest20%Restore 30% HP and mana (guaranteed 1 per 5+ room dungeon)
bossAlways lastPowerful monster from dungeon's boss pool. Completing this finishes the dungeon

Completion Bonus: 50 + 25×level XP and 30 + 15×level gold on top of normal combat rewards.

POST /api/dungeons/enter

Enter a dungeon at the given POI. Must be at the POI location. Resumes progress if dungeon is incomplete.

{ "poiId": "shadowfang-caverns" }

Guards: Not in combat, not traveling, not already in another dungeon, POI type must be dungeon-eligible, 24h cooldown after completion.

POST /api/dungeons/advance

Advance to the next room. Non-combat rooms (trap, treasure, rest) resolve immediately. Combat and boss rooms start a fight — use POST /api/combat/action to battle.

// Trap room response
{ "roomIndex": 1, "roomType": "trap", "status": "cleared",
  "trapType": "poison_dart", "avoided": false,
  "log": ["Poison darts shoot from the walls!", "The trap deals 8 damage!"] }

// Combat room response
{ "roomIndex": 2, "roomType": "combat", "status": "combat_started",
  "message": "A Strong Wolf lurks in the shadows!",
  "monster": { "name": "Strong Wolf", "level": 5, "hp": 30, "maxHp": 30, "type": "beast" } }

// Boss room response
{ "roomIndex": 5, "roomType": "boss", "status": "combat_started",

POST /api/dungeons/leave

Leave the dungeon. Progress is saved — re-enter the same POI to resume. Cannot leave while in combat.

{ "message": "You retreat from the dungeon. Progress saved (3/6 rooms).",
  "progressSaved": true }
Quests
Quest System
Town NPCs offer quests with kill, fetch, and explore objectives. Accept up to 5 active quests, track progress automatically, and return to the quest giver to turn in for rewards.

Objective Types

TypeDescriptionTracking
killKill N of a specific monsterAuto-increments on combat victory
fetchCollect N of a specific itemChecks inventory; items consumed on turn-in
exploreVisit a POI or complete a dungeonAuto-completes on arrival or dungeon completion

Rules

  • Maximum 5 active quests at a time
  • Quests are level-gated — you must be within the quest's level range
  • Some quests have prerequisites (must turn in a previous quest first)
  • You must be at the quest giver's town to accept and turn in
  • Abandoned quests can be re-accepted later
GET /api/quests/available

List quests available at your current town. Filtered by your level and prerequisite completion.

{ "town": "Thornhaven",

  ] }

POST /api/quests/accept

Accept a quest. Must be at the giver's town and within level range.

// Request
{ "questId": "thornhaven-rat-problem" }

// Response
{ "message": "Quest \"Rat Problem\" accepted.",
  "quest": {
    "id": "thornhaven-rat-problem",
    "name": "Rat Problem",
    "status": "active",
    "objectives": [
      { "type": "kill", "target": "giant-rat", "quantity": 5,
        "description": "Kill 5 Giant Rats",
        "current": 0, "required": 5 }
    ],
    "rewards": { "xp": 50, "gold": 20,
      "items": [{ "itemId": "health-potion", "quantity": 3 }] }
  } }
POST /api/quests/turn-in

Turn in a completed quest for rewards. Must be at the giver's town. Fetch items are consumed.

// Request
{ "questId": "thornhaven-rat-problem" }

// Response
{ "message": "Quest \"Rat Problem\" turned in! Rewards: 50 XP, 20 gold, 3x health-potion.",
  "rewards": {
    "xp": 50, "gold": 20,
    "items": [{ "itemId": "health-potion", "quantity": 3 }],
    "leveledUp": false, "newLevel": 1, "newSpells": []
  } }
POST /api/quests/abandon

Abandon an active or completed quest. Progress is lost, but you can re-accept the quest later.

// Request
{ "questId": "thornhaven-wolf-pelts" }

// Response
{ "message": "Quest \"Wolf Pelts\" abandoned. You can accept it again later." }
